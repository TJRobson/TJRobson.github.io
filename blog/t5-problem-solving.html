Problem Solving

05th August, 2016

  • Got blocked on a simple problem.

  • Yes. All the time haha. Every problem seems like a challenge for the past five weeks

  • Solved a problem in an elegant way

  • A small mistake I made recently, was that I had too much space around my blog post titles on my index.html page. I was trying to break down where all the extra space was coming from. So I guess that I was doing a whole lot ‘trying something’, and I got almost nowhere. Then It occurred to me whilst heading to bed that, I may have done something wrong with the HTML. I realised that I had copied and pasted the container element with everything else, so every row had its own container and was taking on those ‘Skeleton’ properties. All I had to do was put all the rows into one container. I realised that a number of problems are going to appear straight forwards once you are aware of the answer.

  • I’ll attempt to describe what these functions do in my own words:

  • .map() takes in an array with transforming expression. Then duplicates a transformed value for all of the original array’s values.

  • .filter() takes in the values of an array along with an boolean expression. It then duplicates those values that meet the expressions conditions or are ‘true’

  • .reduce() takes in values of an array and accumulates or condenses the value into a single value.
